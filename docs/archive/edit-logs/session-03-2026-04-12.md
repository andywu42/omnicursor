# Edit Log — Session 03 (2026-04-12)

## Goal
Fix structural mistake from session 02: hook scripts were placed in `.claude/` which Cursor does not read. Everything belongs in `.cursor/`.

## Root Cause
Session 02 incorrectly applied OmniClaude's directory structure (`.claude/settings.json`) to OmniCursor. Cursor has its own hook system that reads from `.cursor/hooks.json` — it does not read `.claude/settings.json` at all.

## Edits

| # | Action | File | Why |
|---|--------|------|-----|
| 1 | Deleted | `.claude/` (entire directory) | Cursor does not read `.claude/settings.json`. The whole directory was wrong for a Cursor project |
| 2 | Created (dir) | `.cursor/hooks/lib/` | Correct location for shared hook library modules |
| 3 | Created (dir) | `.cursor/hooks/scripts/` | Correct location for hook entry-point scripts |
| 4 | Created | `.cursor/hooks/lib/_common.py` | Moved from `.claude/hooks/lib/`. Updated path resolution (3 levels to repo root instead of 4). `write_context()` uses Cursor's `{"systemMessage": ...}` envelope (not Claude Code's `hookSpecificOutput`) |
| 5 | Created | `.cursor/hooks/lib/pattern_loader.py` | Moved from `.claude/hooks/lib/`. No logic changes |
| 6 | Created | `.cursor/hooks/scripts/user-prompt-submit.py` | Moved from `.claude/hooks/scripts/`. Updated `sys.path` to point to `.cursor/hooks/lib/`. Output format is Cursor's `systemMessage` (correct) |
| 7 | Updated | `.cursor/hooks.json` | `beforeSubmitPrompt` now points to `scripts/user-prompt-submit.py` instead of `on_prompt.py` |

## Key Clarification: `.claude/` vs `.cursor/`

| Directory | Owner | Purpose |
|-----------|-------|---------|
| `.claude/` | Claude Code CLI | Claude Code reads this — irrelevant to Cursor |
| `.cursor/` | Cursor IDE | Cursor reads this — correct home for OmniCursor config |
| `~/.omnicursor/` | OmniCursor runtime | Event logs, sessions, learned patterns — unchanged |

## Output Format Correction

`write_context()` in `_common.py` now correctly outputs Cursor's format:
```python
{"systemMessage": "<markdown content>"}   # correct for Cursor
```
Not Claude Code's format which was used by mistake in session 02:
```python
{"hookSpecificOutput": {"additionalContext": "..."}}  # Claude Code only
```
