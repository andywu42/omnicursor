## What was added

### Delegation Rule
Ported from omniclaude `user-prompt-delegation-rule.sh`.

- Resets per-turn counters (`write_count`, `read_count`, `delegated` flag) in `~/.omnicursor/sessions/{conversation_id}/` at the start of each prompt
- Injects hard rule: "delegate before tool calls if task requires more than 2 steps"
- Threshold hardcoded to 2 (omniclaude reads from config file — OmniCursor has no config system yet)

### Handoff Nudge
Ported from omniclaude `user_prompt_structured_handoff_nudge.sh`.

- Fires **once per session** only (tracked via `~/.omnicursor/sessions/{conversation_id}/handoff_nudge_fired`)
- Only fires if prompt is complex (>50 chars) AND unstructured (no `Task:/Scope:/Done when:` fields, not a `/skill` invocation)
- Injects structured prompt template suggestion

### Why one script instead of three
OmniClaude runs three separate scripts as separate hook entries. Cursor's behavior with multiple `systemMessage` outputs from the same event is undefined — last-write-wins or only-first-used are both possible. A single script producing one combined Markdown block is safer and predictable.
