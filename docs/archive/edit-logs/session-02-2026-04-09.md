## Schema Changes: Cursor → Claude Code

### Input
| Field | Cursor | Claude Code |
|-------|--------|-------------|
| Session ID | `conversation_id` | `sessionId` |
| Prompt | `prompt` | `prompt` (same) |
| Generation ID | `generation_id` | *(not sent)* |

Script reads both `sessionId` and `conversation_id` defensively for compatibility.

### Output
| | Cursor | Claude Code |
|-|--------|-------------|
| Envelope | `{"systemMessage": "..."}` | `{"hookSpecificOutput": {"additionalContext": "..."}}` |
| Content format | HTML comments (`<!-- ... -->`) | Plain Markdown |

### Content format example

**Before (Cursor):**
```
<!-- OmniCursor Agent: debug-intelligence (confidence: 0.95) -->
<!-- Routing reason: Exact trigger: 'debug' -->
<!-- Learned Patterns (from previous sessions): -->
<!-- - [pat-001] Use binary search when debugging list operations -->
```

**After (Claude Code):**
```markdown
## OmniCursor Agent Routing

**Agent:** `debug-intelligence`
**Confidence:** 0.95
**Reason:** Exact trigger: 'debug'

### Learned Patterns
- **[pat-001]** Use binary search when debugging list operations
```

## Logic Unchanged
- Three-strategy scoring (exact → fuzzy → keyword) with `HARD_FLOOR = 0.55`
- `polymorphic-agent` fallback at score 0.0
- Pattern cache warm/stale logic (10-minute threshold)
- Domain derivation from agent name (`agent-` / `omnicursor-` prefix strip)
- Event logging to `~/.omnicursor/events.jsonl`
- Always exits 0, always emits valid JSON
